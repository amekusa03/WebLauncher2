# WebLauncher2

本アプリケーションは、Androidスマートフォンにおいて、特定のWebページをあたかも独立したアプリのように起動・管理するためのアプリケーションである。
WebLauncherのバージョンアップではなく、リライト（焼き直し）との位置づけ。

## 1. 背景と改善点
従来の「5つ制限」をあえて維持しつつ、ユーザーの「動機（買い物、お金、車など）」に基づいたグループ管理（階層構造）を導入することで、利便性を大幅に向上させる。

*   **改善のポイント**:
    *   1次元的な羅列ではなく、「会社名」や「目的」で階層化する。
    *   「あわ茶（お湯呑みアイコン）」という独自のアイデンティティをルートに置く。
    *   最大4階層までのサブ画面展開により、無限に近い登録数を整理された状態で管理する。

## 2. 機能概要
*   ユーザーは「アプリ一覧」または「ホーム画面のショートカット」から起動する。
*   アイコンタップで設定されたURLをブラウザで開く。
*   **「専念モード」**: URL起動後は、管理画面（本アプリ）を自動的に終了（finish）させ、ブラウザ閲覧後にアプリに戻る手間を省く。

## 3. 画面・操作仕様

### 3.1 メイン画面（ルート）
*   固定名称：**あわ茶**
*   固定アイコン：**お湯呑み**
*   5つの登録枠を表示する。

### 3.2 サブ画面（階層）
*   **表示構成**:
    *   上部に「パンくずリスト」を表示。ルートから現在地までの「アイコン」を並べ、現在の階層名のみをテキスト表示する。
    *   パンくず内の過去アイコンをタップすると、その階層へ直接ジャンプできる。
    *   「←」アイコンで1階層上に戻る。
*   **アイコン**: 登録時に指定した16色 × 名前先頭文字の円形アイコン。

### 3.3 アイコンタップ時の挙動
1.  **URLの場合**:
    *   Chrome Custom Tabs（設定による）または標準ブラウザでURLを開く。
    *   **重要**: ブラウザ起動と同時に、本アプリ（Activity）を `finish()` する。
2.  **サブへリンクの場合**: その階層のサブ画面を展開する。
3.  **空白（未登録）の場合**: 登録画面（編集モード）を開く。

### 3.4 長押し時の挙動
*   登録の有無に関わらず、その枠の登録画面を開く。
*   **階層制限**: 現在が4階層目の場合、登録画面において「サブへリンク」への切り替えを無効化（または非表示）とする。

### 3.5 登録・編集画面
*   **入力項目**:
    *   名前（最大64文字）
    *   種別切り替え（URL / サブへのリンク）
    *   URL入力（最大2048文字、http/httpsバリデーションあり）
    *   アイコン色選択（プリセット16色）
    *   Chrome Custom Tabs使用の有無（URL時のみ）
*   **ボタン機能**:
    *   **ホーム画面へ追加**:
        *   現在の設定（名前、色、URL）でショートカットを作成する。
        *   既存のショートカットは上書きせず、押すたびに新しいショートカットをホーム画面に作成する。
    *   **保存**: 同階層内での名前重複チェックを行い、DataStore/Roomへ保存。
    *   **削除**:
        *   URLの場合は即削除。
        *   サブ画面の場合は「配下の登録もすべて削除される」旨の警告ダイアログを表示し、承諾後に再帰的に削除する。

## 4. ホーム画面ショートカットの挙動
*   **ダイレクト起動**: アプリのメイン画面を経由せず、直接URLまたはサブ画面を起動する。
*   **自動更新**: アプリ内で登録内容（URLや色）を変更・保存した際、**すでにホーム画面に配置されている同一IDのショートカットの内容（名前、アイコン、遷移先）をシステム的に可能な範囲で自動更新する。**（ユーザーによる貼り直しを不要にするため）

## 5. 技術仕様
*   **データ管理**: Room Persistence Library（階層構造管理のため、parentIdによる親子関係を定義）。
*   **画面遷移**: Jetpack Compose Navigation。
*   **ブラウザ**: `androidx.browser:browser:1.x.x` (Custom Tabs)。
*   **ショートカット**: `ShortcutManager` API。

## 6. バリデーションルール
*   名前：必須（空白のみ不可）
*   URL：種別がURLの場合必須。`http://` または `https://` で始まること。
*   重複：同一階層（同一parentId）内での名前重複は不可。
